# DECISIONS

## 2026-03-15

### 技术决策
组织架构树主存储从独立树表逐步切换为 `sys_user` 自身字段承载。

原因：
- 用户与组织节点一一对应
- 简化查询链路
- 更便于直接做数据权限过滤
- 有利于将 `parent_user_id / level_no / tree_path` 统一作为核心数据权限基础

## 2026-03-15

### 技术决策
`tree_path` 统一采用带首尾斜杠的物化路径格式，例如 `/1/2/3/`。

原因：
- 便于前缀匹配
- 可避免 `/1/2` 与 `/1/20` 这类路径误匹配问题
- 对节点移动后的路径替换更稳定

## 2026-03-15

### 技术决策
超级管理员身份当前以 `sys_user_role` 中是否存在 `SUPER_ADMIN` 判定。

原因：
- 当前登录、权限、JWT 已按角色表闭环
- 不需要额外引入 `sys_user.super_admin` 字段
- 最小修复成本更低

## 2026-03-15

### 技术决策
组织树操作优先走 `orgtree` 模块，而不是复用通用 `user` 模块直接维护层级关系。

原因：
- 当前树逻辑涉及创建、移动、防循环、路径级联更新
- 放在 `orgtree` 模块更符合职责边界
- 可避免普通用户管理接口误伤树结构

## 2026-03-15

### 技术决策
当前阶段优先做后端最小闭环，再逐步补前端联调和真实数据库验证。

原因：
- 项目当前以功能闭环优先
- 后端先稳定接口与权限边界，更便于后续前端对齐

## 2026-03-16

### 技术决策
`/api/auth/me` 与 `POST /api/auth/login` 统一返回 `LoginVO` 结构，而不是直接暴露 `LoginUser`。

原因：
- 前端登录态字段保持一致，避免 `me` 与 `login` 返回结构漂移
- `LoginUser` 属于鉴权上下文对象，不应直接作为接口返回模型
- 最小改动即可固定 `superAdmin` 等关键字段的对外契约

## 2026-03-16

### 技术决策
前端请求基地址默认使用相对路径 `/api`，开发环境通过 Vite 代理转发到后端，而不是在代码中硬编码 `http://localhost:8080`。

原因：
- 便于 WSL、Windows 宿主机和后续部署环境切换
- 避免浏览器真实联调时被硬编码地址限制
- 让前端构建产物不携带固定后端地址，降低环境耦合

## 2026-03-16

### 技术决策
组织树页面刷新后优先保持当前操作节点选中，而不是每次回退到树的第一个节点。

原因：
- 新增下级和调整上级后更容易直接确认操作结果
- 真实浏览器联调时减少重复定位节点的成本
- 仅修改页面状态逻辑即可形成一个独立的小闭环

## 2026-03-16

### 技术决策
组织树页面在前端先过滤“当前节点自己”和“当前节点的下级”这两类非法移动目标，而不是完全依赖后端报错。

原因：
- 更贴近真实页面操作验证，减少一次无效交互
- 与后端已有防循环规则保持一致
- 修改范围只在页面层，适合作为前端联调阶段的小闭环

## 2026-03-16

### 技术决策
组织树页面在前端拦截“目标上级未变化”的移动提交，不再把这类无意义请求发到后端。

原因：
- 真实页面操作时减少一次无效提交
- 与前端已做的候选过滤形成一致的交互收口
- 只涉及页面状态和按钮可用性，适合作为当前阶段的小闭环

## 2026-03-16

### 技术决策
组织树页面在移动上级弹层中直接展示“当前不可提交的原因”，而不是只禁用按钮不解释原因。

原因：
- 真实浏览器联调时更容易判断当前页面状态是否符合预期
- 能减少用户对“为什么不能提交”的猜测成本
- 只涉及前端展示层，适合作为组织树前端联调阶段的小闭环

## 2026-03-16

### 技术决策
组织树页面的上级链节点支持直接点击切换，而不是只作为静态展示。

原因：
- 真实页面验证时可以沿着上级链快速切换查看节点详情
- 与直属下级卡片已支持点击切换的交互保持一致
- 仅涉及前端页面交互，适合作为当前阶段的小闭环

## 2026-03-16

### 技术决策
组织树页面的直属下级列表在当前节点切换后提供显式高亮反馈，而不是只依赖详情面板变化。

原因：
- 真实页面联调时更容易确认当前定位到哪个节点
- 与树节点高亮、上级链高亮形成一致的反馈方式
- 仅涉及前端展示层，适合作为组织树前端联调阶段的小闭环

## 2026-03-16

### 技术决策
组织树页面为直属下级和上级链补充局部 loading，而不是在这些操作里只依赖全页 loading。

原因：
- 切换节点和局部刷新时反馈更准确，不会让用户误以为整页都在重新加载
- 更贴近真实浏览器联调时对页面状态反馈的观察
- 仅涉及前端页面状态管理，适合作为当前阶段的小闭环

## 2026-03-16

### 技术决策
组织树页面的直属下级和上级链区域在加载时不再同时显示“暂无数据”空态，而是与 loading 互斥展示。

原因：
- 避免页面同时出现互相矛盾的状态提示
- 更符合真实浏览器联调时对页面反馈一致性的要求
- 仅涉及前端展示逻辑，适合作为组织树前端联调阶段的小闭环
