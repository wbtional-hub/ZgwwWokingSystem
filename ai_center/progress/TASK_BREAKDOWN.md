### 目标
确保后端、前端、数据库在当前开发机与 WSL 环境下可稳定运行，并形成最小可联调基础。

### 任务
1. 确认后端可编译、可启动
2. 确认前端可构建、可启动
3. 确认 PostgreSQL 可从 WSL 访问
4. 确认 `wsl` profile 可连接 Windows 宿主机数据库
5. 确认健康检查接口可访问

### 接口
- `GET /api/health`

### 数据库依赖
- PostgreSQL
- 数据库名：`lecture_team`

### 输出
- 后端启动成功
- 前端构建成功
- WSL 到 PostgreSQL 连通
- `/api/health` 返回正常

## 4. 组织架构树

### 目标
基于 `sys_user` 实现金字塔式组织架构最小闭环。

### 任务
1. 补齐 `parent_user_id / level_no / tree_path`
2. 创建下级用户时自动计算层级与路径
3. 查询整棵组织树
4. 查询直属下级
5. 查询上级链
6. 调整上级并级联更新
7. 防循环校验
8. 完成真实 JWT 联调

### 接口
- `POST /api/orgtree/create`
- `GET /api/orgtree/tree`
- `GET /api/orgtree/children/{userId}`
- `GET /api/orgtree/ancestors/{userId}`
- `POST /api/orgtree/move`

### 数据库依赖
- `sys_user`
- 关键字段：
  - `parent_user_id`
  - `level_no`
  - `tree_path`

### 输出
- 组织树真实后端闭环已完成
- 一级、二级、三级节点可创建
- 整树、直属下级、上级链可查
- 节点移动与防循环生效
