# AI能力平台详细画面设计书

## 1. 文档目的

本文档用于定义 AI 能力平台第一阶段各画面的布局、功能区、字段、交互规则和按钮控制要求。

适用范围：

- AI接入区
- 权限中心
- 知识库中心

## 2. 画面设计原则

### 2.1 统一原则

- 延续现有系统风格
- 使用现有移动端/轻后台页面组织方式
- 保持页面简洁、信息清晰、操作集中

### 2.2 权限原则

- 无权限菜单不显示
- 无权限按钮不显示或禁用
- 页面进入前先做权限判断

### 2.3 状态原则

- 列表页显示状态
- 编辑页显示当前状态说明
- 提交中禁止重复操作

### 2.4 审计原则

- 涉及新增、编辑、启停、测试连接、导入的操作，页面应给出明确的结果反馈

## 3. AI接入区

## 3.1 画面 A01 AI接入配置列表

### 画面目标

供管理员统一管理 AI 接入。

### 页面结构

#### 顶部标题区

- 标题：AI接入配置
- 说明：用于录入 AI 提供方 Token、接口地址、默认模型，并进行连通性测试

#### 操作区

按钮：

- 新增接入
- 刷新列表

#### 查询区

字段：

- 关键字
- 状态
- 连接状态

按钮：

- 查询
- 重置

#### 列表区

列表字段：

- 接入编码
- 接入名称
- API地址
- Token掩码
- 默认模型
- 连接状态
- 状态
- 更新时间

行内按钮：

- 编辑
- 启停
- 测试连接

### 交互规则

- Token 只显示掩码
- 测试连接中只锁定当前行按钮
- 停用状态的接入不可作为后续默认接入使用

## 3.2 画面 A02 AI接入配置编辑页

### 页面结构

#### 基础信息区

字段：

- 接入编码
- 接入名称
- API Base URL
- API Token
- 默认模型
- 备注

#### 按钮区

- 保存
- 清空
- 返回

### 交互规则

- 编辑状态不展示明文旧 Token
- 若未重新输入 Token，则沿用原 Token
- 保存前校验编码、名称、URL 必填

## 5.2 画面 K02 知识库详情页

### 页面目标

查看一个知识库内的分类、文档、知识块。

### 页面布局

左右分栏或上下分区结构：

#### 区域一 分类树

字段：

- 分类名称
- 状态

按钮：

- 新增分类
- 编辑分类

#### 区域二 文档列表

字段：

- 文档标题
- 文档类型
- 地区
- 政策级别
- 生效时间
- 失效时间
- 解析状态
- 状态

按钮：

- 导入文档
- 查看文档详情

#### 区域三 文档详情与知识块

字段：

- 标题路径
- Chunk序号
- 内容
- 关键词
- 长度

### 交互规则

- 点击分类树联动刷新文档列表
- 点击文档联动展示知识块
- 已失效文档应有状态提示

## 5.3 画面 K03 文档导入页

### 页面目标

完成 Word 文档导入、元信息录入和导入结果反馈。

### 页面结构

#### 基础表单区

字段：

- 知识库
- 分类
- 上传文件
- 地区
- 政策级别
- 生效时间
- 失效时间
- 摘要
- 关键词

#### 操作区

- 开始导入
- 清空
- 返回

#### 结果区

字段：

- 导入任务ID
- 文件名
- 状态
- 总Chunk数
- 成功Chunk数
- 错误信息

### 交互规则

- 仅允许上传 Word
- 未选择知识库不可提交
- 导入成功后可跳转到知识库详情页

## 5.4 画面 K04 知识检索页

### 页面目标

让授权用户检索知识内容。

### 页面结构

#### 查询区

字段：

- 知识库
- 分类
- 关键词
- 地区
- 是否只看有效期内
- TopN

#### 结果区

字段：

- 文档标题
- 地区
- 标题路径
- 片段摘要
- Chunk序号

### 交互规则

- 无权限用户不可进入
- 结果应优先返回命中文档和核心片段

## 6. 菜单与权限联动

## 6.1 菜单建议

- AI接入区
- 知识库中心
- AI权限配置

## 6.2 权限联动规则

### AI接入区

- 仅管理员显示

### 知识库中心

- 管理页：管理员/知识管理员
- 检索页：按知识库授权开放

### 权限中心

- 仅管理员显示

## 7. 通用交互要求

### 7.1 列表页

- 支持刷新
- 支持空态展示
- 支持当前筛选摘要

### 7.2 弹层页

- 支持保存
- 支持重置
- 有明确校验提示

### 7.3 异步操作

- 需有 loading 状态
- 防止重复提交
- 操作成功给出 toast
- 操作失败给出明确错误

## 8. 首期重点画面

优先级排序：

1. K01 知识库管理页
2. K03 文档导入页
3. K02 知识库详情页
4. A01 AI接入配置列表
5. A02 AI接入配置编辑页
6. P01 AI能力授权页
7. P02 知识库授权页
8. K04 知识检索页

## 9. 结论

第一期画面设计重点是把底座搭起来：

- AI 可以接
- 知识可以导
- 权限可以控
- 知识可以查

在此基础上，第二期再进入技能训练、验证和工作台。
